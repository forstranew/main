JSP 프로그래밍 시험 (36~60번)

36. JSP에서 request 기본 객체가 제공하는 기능은?
request 객체는 클라이언트가 보낸 요청 데이터(파라미터, 헤더 등)를 읽을 때 사용된다.

37. JSP 선언문(<%! %>)의 특징은?
선언문은 JSP 내에서 멤버 변수와 메서드를 정의할 때 사용된다.

38. session 기본 객체의 주요 용도는?
session 객체는 사용자별로 데이터(로그인 정보 등)를 유지할 때 사용된다.

39. JSP include 지시자의 특징은?
include 지시자는 JSP 변환 시 파일이 그대로 합쳐진다.

40. JSP 페이지 생명주기에서 가장 먼저 호출되는 메서드는?
jspInit()은 JSP가 최초 로딩될 때 단 한 번 실행된다.

41. JSP 스크립트릿(<% %>)의 역할은?
스크립트릿은 JSP 내에서 Java 실행문을 작성할 때 사용된다.

42. JSP에서 페이지 이동 시 request 객체를 유지하려면 사용하는 방식은?
forward()는 request와 response 객체를 그대로 다음 페이지로 전달한다.

43. JSP EL(Expression Language)의 특징은?
EL은 ${param.id} 처럼 간단히 객체나 파라미터 값을 접근할 수 있다.

44. JSP에서 사용되는 out 객체의 기능은?
out 객체는 JSP에서 클라이언트로 출력되는 내용을 버퍼에 저장한다.

45. JSP에서 page 지시자의 contentType 속성의 역할은?
contentType은 브라우저가 어떤 형식의 문서인지 인식하게 하는 MIME 타입을 지정한다.

46. JSP에서 response 객체의 역할은?
response 객체는 클라이언트에게 전송되는 응답을 구성한다.

47. JSP의 session 기본 객체가 생성되는 시점은?
session 객체는 클라이언트가 처음 JSP에 접근할 때 자동 생성된다.

48. JSP에서 에러 페이지를 지정할 때 사용하는 속성은?
errorPage 속성을 사용하여 오류 발생 시 이동할 JSP를 지정한다.

49. JSP의 autoFlush 속성이 true일 때 의미는?
autoFlush=true는 버퍼가 꽉 차면 자동으로 클라이언트로 전송한다는 의미이다.

50. JSP에서 Tag Library를 사용할 때 필요한 선언은?
taglib 지시자를 사용해야 사용자 정의 태그나 JSTL을 사용할 수 있다.

51. JSP의 EL에서 param.name 의 역할은?
${param.xxx}는 request 파라미터 값을 읽는다.

52. JSP의 쿠키 생성 시 사용하는 객체는?
쿠키는 Cookie 클래스를 사용해 생성한다.

53. JSP에서 POST 방식 데이터의 인코딩 설정은 언제 적용해야 하는가?
POST 방식은 request.getParameter()를 호출하기 전에 setCharacterEncoding()을 적용해야 한다.

54. JSP와 Servlet의 관계를 가장 잘 설명한 것은?
JSP는 내부적으로 Servlet 코드로 변환된 후 실행된다.

55. JSP에서 scope 중 가장 좁은 범위는?
page scope는 현재 JSP 페이지에서만 유효하며 가장 좁은 범위이다.

56. JSP에서 세션을 비활성화하려면 page 지시자에서 어떤 속성을 설정해야 하는가?
session=false 로 설정하면 JSP는 세션 객체를 생성하지 않는다.

57. JSP에서 null 값을 방지하기 위해 사용되는 EL 연산자는?
empty는 값이 null 또는 빈 문자열인지 확인하는 데 사용된다.

58. JSP에서 requestDispatcher.forward()의 특징은?
forward()는 요청 정보를 그대로 유지하며 서버 내부에서 페이지를 이동한다.

59. JSP에서 sendRedirect()의 특징은?
sendRedirect()는 브라우저에게 새로운 URL로 재요청하라고 지시하는 방식이다.

60. JSP에서 JSTL의 Core 라이브러리를 사용하기 위해 필요한 URI는?
JSTL Core 라이브러리는 다음 URI를 사용한다: http://java.sun.com/jsp/jstl/core
