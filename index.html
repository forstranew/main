문제 36: 다음이 설명하는 HTTP 요청 방식은 무엇인가?
원하는 방식으로 인코딩된 데이터를 요청 메시지의 몸체에 포함하여 전송하면서 자원을 요청하는 경우 사용됨
① GET
② HEAD
③ POST
④ FTP
POST 방식은 데이터를 HTTP 요청 메시지의 본문(body)에 포함하여 전송하는 방식입니다. URL에 데이터가 노출되지 않으며, 대용량이나 민감한 정보 전송 시 사용됩니다.
문제 37: "Eclipse를 이용한 웹 프로젝트의 배포"가 의미하는 것은?
① 웹 프로젝트 폴더를 압축 파일로 만드는 것이다.
② 개발한 웹 프로젝트를 웹 컨테이너에 내보내는 것이다.
③ 웹 컨테이너를 서버로 등록시키는 것이다.
④ 버퍼의 내용을 일차적으로 클라이언트에 보내는 것이다.
배포(Deploy)는 개발 완료된 웹 애플리케이션을 실행할 수 있도록 웹 컨테이너(Tomcat 등)에 전달하는 작업입니다.
문제 38: JSP 페이지에서 메소드나 변수를 선언하기 위한 태그는?
① <% ... %>
② <%@ ... %>
③ <%! ... %>
④ <%# ... %>
<%! ... %>는 JSP 페이지에서 멤버 변수나 메소드를 선언할 때 사용됩니다. 선언된 변수와 메소드는 서블릿 클래스의 멤버로 변환되어 모든 요청에서 공유됩니다.
문제 39: 서블릿 클래스에서 doGet() 메서드의 인자는 무엇인가?

③ request와 response

doGet() 메서드는 HttpServletRequest와 HttpServletResponse 두 개의 인자를 받습니다.

<%@ page _____="true" %>

<% Cookie cookie = new Cookie("name", "value"); response.addCookie(cookie); %>

문제 55: JSTL의 반복 태그로 잘못된 것은?
① <c:forEach>
② <c:forTokens>
③ <c:forAll>
④ <c:repeat>

문제 56: JSTL 조건 태그 중 사용되지 않는 것은?
① <c:if>

<c:case>는 JSTL에 없는 태그이며, 조건 분기에는 <c:if>, <c:choose>, <c:when>, <c:otherwise>를 사용합니다.
