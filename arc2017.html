컴퓨터 구조 기출문제 상세 해설 (50제)

🔵 문제 1

다음 중 컴퓨터 명령어를 구성하는 연산 코드(OP code field)에 대한 설명으로 가장 적절한 것은?

① 기억장치 번지 혹은 처리장치 레지스터를 선택하기 위한 필드
② 명령어가 수행할 연산을 지정하는 필드이다.
③ 명령어의 주소 지정 방식을 결정하는 필드이다.
④ 인터럽트 요청을 처리할 제어 필드이다.

✅ 정답: ②

상세 해설:
명령어는 크게 연산 코드(Opcode)와 오퍼랜드(Operand)로 나뉩니다.
Opcode 필드는 CPU가 어떤 연산을 수행해야 하는지를 명시합니다. 예: ADD, SUB, LOAD, STORE, JMP 등.
나머지 필드는 주소 부분이나 모드 비트 등에 해당하며, Opcode가 아닙니다.

🔵 문제 2

다음 컴퓨터 명령어에 관한 설명으로 적절한 것은?

① 2-주소 컴퓨터 명령어이다.
② 데이터 전송을 위한 컴퓨터 명령어이다.
③ 컴퓨터의 내부 구조로 볼 때 누산기를 이용하는 컴퓨터 명령어이다.
④ 컴퓨터의 내부 구조로 볼 때 다중 레지스터를 이용하는 컴퓨터 명령어이다.

✅ 정답: ③

상세 해설:
주어진 프로그램(LOAD A, ADD B 등)은 명시적인 피연산자가 하나만 보이고, 나머지 하나는 누산기(AC)가 암묵적으로 사용됩니다.
→ 이는 전형적인 1-주소 명령어 형식(누산기 기반 아키텍처)입니다.
2-주소 방식은 두 개의 명시적 주소를 가지며, 다중 레지스터 방식은 범용 레지스터를 여러 개 사용합니다.

🔵 문제 3

위 프로그램은 다음 중 어느 수식을 계산하는 것인가?

① X = (A + B) × C
② X = C × (A + B)
③ X = (A + B) / C
④ X = (A × B) + C

✅ 정답: ②

단계별 수행 과정:
1. LOAD A → AC = A
2. ADD B → AC = A + B
3. STORE X → M[X] = A + B
4. LOAD C → AC = C
5. MUL X → AC = C × (A + B)
6. STORE X → X = C × (A + B)
→ 최종 결과는 C × (A + B)입니다.

🔵 문제 35

다음 중 병렬 처리를 수행하는 시스템이 아닌 것은?

① pipeline processor
② array processor
③ multiple processor
④ I/O processor

✅ 정답: ④

상세 해설:
• Pipeline Processor: 명령어를 겹쳐서 실행 (명령어 수준 병렬성)
• Array Processor: 벡터/배열 데이터를 동시에 처리 (SIMD)
• Multiple Processor: 여러 CPU가 동시에 작업 (멀티프로세서)
• I/O Processor: 입출력 전담 프로세서 → 병렬 처리 목적이 아님
